# Security Policy

## Supported Versions

This project is under active development. Security updates are applied to the latest version on the `main` branch.


## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, report the issue privately via email:

📧 laubeja7@gmail.com


## What to Include

Please include:
- A clear description of the issue
- Steps to reproduce (if applicable)
- Potential impact
- Any suggested fixes (optional)

## Response Time

We aim to respond to all security reports within **72 hours**.

## Scope

This project is a **local desktop application** and does not involve:
- Remote servers
- User accounts
- Online data storage

Security concerns are primarily related to:
- Local file handling
- Dependency vulnerabilities
- Safe execution of code

## Dependency Security

We aim to keep dependencies up to date. If you identify a vulnerable dependency, please report it.


## Responsible Disclosure

Please:
- Avoid publicly disclosing the issue before it is fixed
- Allow time for a resolution

We will address valid issues as quickly as possible.

## Acknowledgement

We appreciate responsible disclosure and contributions that improve the security of this project.
